# 全景图像拼接批量实验结果中文分析报告

项目题目：《基于 OpenStitching 的全景图像拼接与融合优化系统设计》

本报告根据已经运行完成的实验结果自动整理，主要解释 `outputs/` 目录下生成的 CSV 表格、对比图和报告素材。实验命令为：

```bash
python run_all_experiments.py --data data --output outputs --resize-width 1200
```

## 1. 这个实验到底做了什么

本次实验不是只拼一张全景图，而是把不同类型的图片组批量跑了一遍。每一组图片都会经历三类实验：

1. **baseline 基础版**

   直接使用 OpenStitching 的默认拼接流程。它的作用是回答一个最基础的问题：

   > 这组图片在默认算法下能不能拼起来？

2. **improved 改进版**

   在基础拼接前后加入一些图像处理步骤，例如缩放、曝光补偿、自动裁剪黑边。它的作用是回答：

   > 经过预处理和后处理之后，结果是否更稳定、更适合展示？

3. **feature_experiment 算法加深版**

   对 SIFT、ORB、AKAZE 三种特征点算法进行对比，统计关键点数量、匹配点数量、RANSAC 内点数量、内点率和运行时间。它的作用是回答：

   > 为什么有些图能拼起来？不同特征算法在不同场景下有什么差别？

简单理解：

| 模式 | 关注点 | 报告中怎么写 |
|---|---|---|
| baseline | 能不能拼 | 验证 OpenStitching 默认流程 |
| improved | 拼得好不好看 | 展示曝光补偿、裁剪黑边等优化效果 |
| feature_experiment | 为什么能拼/为什么失败 | 分析 SIFT、ORB、AKAZE 的匹配和 RANSAC 表现 |

### 3.2 baseline 失败的图片组

baseline 失败组如下：

```text
07_large_parallax/set_large_parallax_1
07_large_parallax/set_large_parallax_3
09_failure_cases/set_failure_1
10_self_captured_demo/set_self_demo_1
10_self_captured_demo/set_self_demo_2
10_self_captured_demo/set_self_demo_3
```

失败原因主要有两类：

1. **图片数量不足**

   `10_self_captured_demo` 是空占位目录，所以程序提示：

   ```text
   Need at least two images.
   ```

   这类失败不用担心，后续你自己放入 2-6 张有重叠的照片即可。

2. **匹配置信度不足**

   对于大视差或失败案例，OpenStitching 提示类似：

   ```text
   No match exceeds the given confidence threshold.
   ```

   中文解释就是：

   > 程序找到了特征点，但相邻图片之间可靠匹配不够多，无法确定这些图是否属于同一个全景。

这很适合写进报告的“失败原因分析”部分。

## 6. SIFT / ORB / AKAZE 特征算法对比

特征算法对比主要看几个指标：

| 指标 | 含义 |
|---|---|
| keypoints | 检测到的特征点数量 |
| raw matches | 初始匹配数量 |
| good matches | 经过 ratio test 筛选后的较好匹配 |
| RANSAC inliers | 经过 RANSAC 保留下来的可靠匹配 |
| inlier ratio | 内点率，越高说明匹配越可靠 |
| runtime | 耗时，越低越快 |

### 6.1 三种算法整体表现

| 算法 | 平均内点率 | good matches 总数 | RANSAC 内点总数 | 总耗时 |
|---|---:|---:|---:|---:|
| SIFT | 0.7819 | 53190 | 43441 | 71.45 s |
| ORB | 0.7672 | 50319 | 40611 | 76.97 s |
| AKAZE | 0.7881 | 50732 | 41860 | 57.25 s |

从本次实验结果看：

1. **AKAZE 平均内点率最高**

   AKAZE 的平均内点率为 0.7881，是三者中最高的。

2. **SIFT 匹配数量最多**

   SIFT 的 good matches 和 RANSAC 内点总数最高，说明它在很多场景下能找到更丰富的可靠匹配。

3. **AKAZE 总耗时最低**

   AKAZE 总耗时约 57.25 秒，比 SIFT 和 ORB 都低。

所以，本次实验可以得出一个比较稳妥的结论：

> AKAZE 在本批数据上综合表现最好；SIFT 稳定性强、匹配数量多；ORB 可作为轻量算法对比，但本次整体内点率略低。

## 7. 不同类型图像下的算法表现

### 7.1 景观类

景观类纹理丰富，树木、道路、建筑边缘较多，三种算法都能找到较多特征点。

在这类图像中：

- ORB、SIFT、AKAZE 都能工作。
- 特征点充足，拼接成功率高。
- 适合作为“基础成功案例”展示。

推荐展示：

```text
01_landscape_rich_texture/set_landscape_1
```

### 7.2 建筑类

建筑类有很多直线、角点和边缘，比如楼体、窗户、道路。

实验结果显示：

- SIFT 和 AKAZE 表现较稳定。
- 建筑类虽然特征丰富，但窗户、墙砖等重复结构可能导致误匹配。
- RANSAC 的作用很关键，它可以剔除错误匹配。

### 7.3 地图/卫星类

地图/卫星类场景接近平面，比较符合 Homography 模型，因此整体拼接较容易成功。

但是地图类也可能有重复线条、文字或规则图案，因此仍可能产生误匹配。

### 7.4 室内弱纹理类

室内弱纹理类通常特征点较少，比如墙面、桌面、走廊。

本次数据中仍然能够拼接成功，但报告中可以这样分析：

> 弱纹理场景下，特征点数量减少，算法更依赖少量可靠角点。如果图像更平滑或重叠区域更少，拼接会更容易失败。

### 7.5 低曝光/曝光差异类

低曝光类适合展示 improved 的意义。

因为 Gamma、CLAHE、曝光补偿的作用就是：

- 提高暗部可见性
- 减少不同图片之间的亮度差
- 帮助特征点检测和视觉融合

报告中可以写：

> 对曝光差异明显的图像，改进版通过亮度处理和曝光补偿，使输入图像亮度更一致，从而改善拼接后的视觉连续性。

### 7.6 重复纹理类

重复纹理类很适合做算法分析。

因为这类图像可能出现：

- 匹配点数量很多
- 但错误匹配也很多
- 单纯看匹配数量不可靠
- 需要看 RANSAC 内点率

报告中可以写：

> 重复纹理场景说明，匹配点数量多不一定代表拼接可靠，RANSAC 内点率更能反映匹配质量。

### 7.7 大视差类

大视差类是本次实验中最容易失败的类型之一。

原因是全景拼接通常假设图像之间可以用 Homography 描述，而 Homography 更适合：

- 相机绕光心旋转
- 或者拍摄平面场景

如果近景物体很多，拍摄位置变化明显，就会产生大视差。此时单个 Homography 很难同时对齐远景和近景，因此容易出现：

- 拼接失败
- 重影
- 局部错位

推荐作为失败案例：

```text
07_large_parallax/set_large_parallax_3
```

### 7.8 多图大视野类

多图大视野类用于测试多张图片连续拼接能力。

这类数据可以展示：

- 多图拼接能力
- 拼接范围扩大
- 累积误差
- 自动裁剪黑边效果

推荐展示：

```text
08_many_images_wide_view/set_many_images_2
```

### 7.9 失败案例类

失败案例类本来就是为了分析算法不足而设计的。

推荐展示：

```text
09_failure_cases/set_failure_1
```

它适合说明：

- 重叠区域不足会导致匹配失败
- 纹理不足会导致特征点不稳定
- 即使有预处理，也不能解决所有几何问题

## 9. 可以直接写进报告的总结

本项目构建了一个基于 OpenStitching 和 OpenCV 的全景图像拼接批量实验系统。系统包含基础版、改进版和算法加深版三种模式。基础版用于验证 OpenStitching 默认流程的拼接能力；改进版加入图像预处理、曝光补偿和自动裁剪黑边，用于提升结果的视觉质量；算法加深版通过 SIFT、ORB、AKAZE 三种特征算法对比，分析特征检测、匹配和 RANSAC 内点筛选对拼接结果的影响。

实验共测试 33 组图片。baseline 模式成功 27 组，improved 模式成功 28 组，说明预处理和后处理能够在部分困难场景中改善拼接效果。对于大多数纹理丰富、建筑、地图和多图连续场景，OpenStitching 能够稳定完成拼接；而在大视差、重叠不足和失败案例中，算法容易出现匹配置信度不足或拼接失败。

特征算法对比结果表明，AKAZE 在本批数据上的平均内点率最高，且整体耗时最低，综合表现较好；SIFT 的 good matches 和 RANSAC 内点总数最高，说明其匹配稳定性较强；ORB 可以作为轻量级算法进行对比，但本次实验中整体内点率略低。失败案例说明，图像拼接不仅依赖特征点数量，还受到重叠区域、视差大小、纹理丰富程度和 Homography 模型假设的影响。
